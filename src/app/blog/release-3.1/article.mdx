  # New LoadingIsland for TextInput

  Using the new `LoadingIsland` you can show a loading spinner inside the `TextInput` component. This feature is useful when
  you need to show that the component is in loading state. This is necessary when you implement a search component or
  when you need to show that the component is waiting for some data.

  <CodeBlock>{`import { TextInput } from "altrone-ui";

return <TextInput placeholder="Input with spinner">
  <TextInput.LoadingIsland />
</TextInput>`}</CodeBlock>
</Text.Section>

  # Migration to v3.1

  - due to conflicts with ARIA tags we decided to mark `role` prop in **Button** as deprecated (it will be removed in 4.0). Currently you can replace this prop with new `severity` prop and use new `ariaRole` prop to override default role of the component.
  - due to conflicts with ARIA tags we decided to mark `role` prop in **Message** as deprecated (it will be removed in 4.0). Currently you can replace this prop with new `severity` prop and use new `ariaRole` prop to override default role of the component.
  - due to conflicts with ARIA tags we decided to mark `role` prop in **Text.Heading** as deprecated (it will be removed in 4.0). Currently you can replace this prop with new `variant` prop and use new `ariaRole` prop to override default role of the component.
</Text.Section>
